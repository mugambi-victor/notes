What is MONGODB

mongodb is a no-sql database program.

It uses document-oriented data model , which means data is stored in Json-like documents.

These documets are flexible in structure, allowing fields to vary from document to document within a collection.

Key features of mongo db
  1. document-oriented: MongoDb stores data in JSON-like documents.
  1. schemaless: unlike traditional databases, MongoDb does not require a predefined schema. This flexibility helps in environments where data structure may evolve overtime.
  1. High perfomance: MongoDb’s architecture and indexing capabilities provide fast read and write operations, making it suitable for applications with high throughput requirements.
  1. scalability: MongoDB is designed to scale out horizontally, allowing for high performance across large datasets and high traffic volumes.
  1. Rich Query Language: MongoDB supports a powerful query language for various operations like filtering, sorting and aggregation.

Unit 01: Getting Started with MongoDB Atlas, the Developer Data Platform.

  1. Atlas is a comprehensive data platform. It allows users to deploy, manage and scale mongoDb databases with ease in the cloud environment.
  1. Atlas enables database replication to enhance redundancy and availability incase of server failure.
  1. Data on Atlas can be managed using the Atlas CLI or Administration API
  1. Atlas search: enables you to build rich search experience on top of your data on the cloud.
  1. Atlas devicing: Allows you to keep data in sync on cloud and mobile devices or other edge devices.
  1. Atlas data lake: lets you automatically extract data from your atlas databases to your iwn low cost, fully managed data lake environment. Data lake uses a format that’s optimized for analytical queries.
  1. Atlas data federation: It enables you to smoothly query, transform, aggregate, and write data from Atlas databases, Atlas Data Lakes, or your own cloud object storage environments.
  1. Atlas Chart: This is an integrated data visualization tool. It enables you to effortlessly create, share, and embed detailed dashboards from your data stored in the cloud.
  1. Atlas App services: offers a set of capabilities that allow you to build faster. It includes a data API, which enables you to read and write data in ATLAS with standard HTTP requests. It also includes a GraphQL API, functions and triggers to enable you to easily create applications with event driven architectures.

Unit 03: Connecting to a MongoDB Database

  1. mongo db stores data in structures called documents which are similar to json objects. example:
{
"_id": 1,
"name": "AC3 Phone",
"colors" : ["black", "silver"],
"price" : 200,
"available" : true
}
  1. While mongo db data are displayed in json format they are stored in the database using a format called BSON(Binary json).
    BSON
     is an extension of JSON providing additional features that MongoDB can leverage. Also supports additional datatypes not available in JSON.
  1. Every document requires an _id field, which acts as a primary key. If _id field is not specified, mongodb automatically generates an ObjectId for the _id field.
  1. Mongodb has flexible schema. eg if you had 3 fields in your document and during develpoment you remember you have to include a forth one, you just need to update the classes to accomodate this new field without explicitly having to add the column name like in relational dbs.
